<span class="s1">&lt;p&gt;</span><span class="s0">Getting pygame imported and initialized is a very simple process. It is also</span>
<span class="s0">flexible enough to give you control over what is happening. Pygame is a</span>
<span class="s0">collection of different modules in a single python package. Some of the</span>
<span class="s0">modules are written in C, and some are written in python. Some modules</span>
<span class="s0">are also optional, and might not always be present.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">This is just a quick introduction on what is going on when you import pygame.</span>
<span class="s0">For a clearer explanation definitely see the pygame examples.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;section </span><span class="s2">id</span><span class="s3">=&quot;import&quot;</span><span class="s1">&gt;</span>
<span class="s1">&lt;h3&gt;</span><span class="s0">Import</span><span class="s1">&lt;a </span><span class="s2">class</span><span class="s3">=&quot;headerlink&quot; </span><span class="s2">href</span><span class="s3">=&quot;#import&quot; </span><span class="s2">title</span><span class="s3">=&quot;Permalink to this heading&quot;</span><span class="s1">&gt;</span><span class="s0">¶</span><span class="s1">&lt;/a&gt;&lt;/h3&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">First we must import the pygame package. Since pygame version 1.4 this</span>
<span class="s0">has been updated to be much easier. Most games will import all of pygame like this.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;div </span><span class="s2">class</span><span class="s3">=&quot;highlight-default notranslate&quot;</span><span class="s1">&gt;&lt;div </span><span class="s2">class</span><span class="s3">=&quot;highlight&quot;</span><span class="s1">&gt;&lt;pre&gt;&lt;span&gt;&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;kn&quot;</span><span class="s1">&gt;</span><span class="s0">import</span><span class="s1">&lt;/span&gt; &lt;span </span><span class="s2">class</span><span class="s3">=&quot;nn&quot;</span><span class="s1">&gt;</span><span class="s0">pygame</span><span class="s1">&lt;/span&gt;</span>
<span class="s1">&lt;span </span><span class="s2">class</span><span class="s3">=&quot;kn&quot;</span><span class="s1">&gt;</span><span class="s0">from</span><span class="s1">&lt;/span&gt; &lt;span </span><span class="s2">class</span><span class="s3">=&quot;nn&quot;</span><span class="s1">&gt;</span><span class="s0">pygame.locals</span><span class="s1">&lt;/span&gt; &lt;span </span><span class="s2">class</span><span class="s3">=&quot;kn&quot;</span><span class="s1">&gt;</span><span class="s0">import</span><span class="s1">&lt;/span&gt; &lt;span </span><span class="s2">class</span><span class="s3">=&quot;o&quot;</span><span class="s1">&gt;</span><span class="s0">*</span><span class="s1">&lt;/span&gt;</span>
<span class="s1">&lt;/pre&gt;&lt;/div&gt;</span>
<span class="s1">&lt;/div&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">The first line here is the only necessary one. It imports all the available pygame</span>
<span class="s0">modules into the pygame package. The second line is optional, and puts a limited</span>
<span class="s0">set of constants and functions into the global namespace of your script.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">An important thing to keep in mind is that several pygame modules are optional.</span>
<span class="s0">For example, one of these is the font module. When  you </span><span class="s4">&amp;quot;</span><span class="s0">import pygame</span><span class="s4">&amp;quot;</span><span class="s0">, pygame</span>
<span class="s0">will check to see if the font module is available. If the font module is available</span>
<span class="s0">it will be imported as </span><span class="s4">&amp;quot;</span><span class="s0">pygame.font</span><span class="s4">&amp;quot;</span><span class="s0">. If the module is not available, </span><span class="s4">&amp;quot;</span><span class="s0">pygame.font</span><span class="s4">&amp;quot;</span>
<span class="s0">will be set to None. This makes it fairly easy to later on test if the font module is available.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;/section&gt;</span>
<span class="s1">&lt;section </span><span class="s2">id</span><span class="s3">=&quot;init&quot;</span><span class="s1">&gt;</span>
<span class="s1">&lt;h3&gt;</span><span class="s0">Init</span><span class="s1">&lt;a </span><span class="s2">class</span><span class="s3">=&quot;headerlink&quot; </span><span class="s2">href</span><span class="s3">=&quot;#init&quot; </span><span class="s2">title</span><span class="s3">=&quot;Permalink to this heading&quot;</span><span class="s1">&gt;</span><span class="s0">¶</span><span class="s1">&lt;/a&gt;&lt;/h3&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">Before you can do much with pygame, you will need to initialize it. The most common</span>
<span class="s0">way to do this is just make one call.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;div </span><span class="s2">class</span><span class="s3">=&quot;highlight-default notranslate&quot;</span><span class="s1">&gt;&lt;div </span><span class="s2">class</span><span class="s3">=&quot;highlight&quot;</span><span class="s1">&gt;&lt;pre&gt;&lt;span&gt;&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;n&quot;</span><span class="s1">&gt;</span><span class="s0">pygame</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;o&quot;</span><span class="s1">&gt;</span><span class="s0">.</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;n&quot;</span><span class="s1">&gt;</span><span class="s0">init</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;p&quot;</span><span class="s1">&gt;</span><span class="s0">()</span><span class="s1">&lt;/span&gt;</span>
<span class="s1">&lt;/pre&gt;&lt;/div&gt;</span>
<span class="s1">&lt;/div&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">This will attempt to initialize all the pygame modules for you. Not all pygame modules</span>
<span class="s0">need to be initialized, but this will automatically initialize the ones that do. You can</span>
<span class="s0">also easily initialize each pygame module by hand. For example to only initialize the</span>
<span class="s0">font module you would just call.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;div </span><span class="s2">class</span><span class="s3">=&quot;highlight-default notranslate&quot;</span><span class="s1">&gt;&lt;div </span><span class="s2">class</span><span class="s3">=&quot;highlight&quot;</span><span class="s1">&gt;&lt;pre&gt;&lt;span&gt;&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;n&quot;</span><span class="s1">&gt;</span><span class="s0">pygame</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;o&quot;</span><span class="s1">&gt;</span><span class="s0">.</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;n&quot;</span><span class="s1">&gt;</span><span class="s0">font</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;o&quot;</span><span class="s1">&gt;</span><span class="s0">.</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;n&quot;</span><span class="s1">&gt;</span><span class="s0">init</span><span class="s1">&lt;/span&gt;&lt;span </span><span class="s2">class</span><span class="s3">=&quot;p&quot;</span><span class="s1">&gt;</span><span class="s0">()</span><span class="s1">&lt;/span&gt;</span>
<span class="s1">&lt;/pre&gt;&lt;/div&gt;</span>
<span class="s1">&lt;/div&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">Note that if there is an error when you initialize with </span><span class="s4">&amp;quot;</span><span class="s0">pygame.init()</span><span class="s4">&amp;quot;</span><span class="s0">, it will silently fail.</span>
<span class="s0">When hand initializing modules like this, any errors will raise an exception. Any</span>
<span class="s0">modules that must be initialized also have a </span><span class="s4">&amp;quot;</span><span class="s0">get_init()</span><span class="s4">&amp;quot; </span><span class="s0">function, which will return true</span>
<span class="s0">if the module has been initialized.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">It is safe to call the init() function for any module more than once.</span><span class="s1">&lt;/p&gt;</span>
<span class="s1">&lt;/section&gt;</span>
<span class="s1">&lt;section </span><span class="s2">id</span><span class="s3">=&quot;quit&quot;</span><span class="s1">&gt;</span>
<span class="s1">&lt;h3&gt;</span><span class="s0">Quit</span><span class="s1">&lt;a </span><span class="s2">class</span><span class="s3">=&quot;headerlink&quot; </span><span class="s2">href</span><span class="s3">=&quot;#quit&quot; </span><span class="s2">title</span><span class="s3">=&quot;Permalink to this heading&quot;</span><span class="s1">&gt;</span><span class="s0">¶</span><span class="s1">&lt;/a&gt;&lt;/h3&gt;</span>
<span class="s1">&lt;p&gt;</span><span class="s0">Modules that are initialized also usually have a quit() function that will clean up.</span>
<span class="s0">There is no need to explicitly call these, as pygame will cleanly quit all the</span>
<span class="s0">initialized modules when python finishes.</span><span class="s1">&lt;/p&gt;</span>

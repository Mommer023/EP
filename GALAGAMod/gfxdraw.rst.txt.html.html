<span class="s0">:mod:`pygame.gfxdraw`</span>
<span class="s0">=====================</span>

<span class="s0">.. module:: pygame.gfxdraw</span>
   <span class="s0">:synopsis: pygame module for drawing shapes</span>

<span class="s0">| :sl:`pygame module for drawing shapes`</span>

<span class="s0">**EXPERIMENTAL!**: This API may change or disappear in later pygame releases. If</span>
<span class="s0">you use this, your code may break with the next pygame release.</span>

<span class="s0">The pygame package does not import gfxdraw automatically when loaded, so it</span>
<span class="s0">must imported explicitly to be used.</span>

<span class="s0">::</span>

    <span class="s0">import pygame</span>
    <span class="s0">import pygame.gfxdraw</span>

<span class="s0">For all functions the arguments are strictly positional and integers are</span>
<span class="s0">accepted for coordinates and radii. The ``color`` argument can be one of the</span>
<span class="s0">following formats:</span>

   <span class="s0">- a :mod:`pygame.Color` object</span>
   <span class="s0">- an ``(RGB)`` triplet (tuple/list)</span>
   <span class="s0">- an ``(RGBA)`` quadruplet (tuple/list)</span>

<span class="s0">The functions :meth:`rectangle` and :meth:`box` will accept any ``(x, y, w, h)``</span>
<span class="s0">sequence for their ``rect`` argument, though :mod:`pygame.Rect` instances are</span>
<span class="s0">preferred.</span>

<span class="s0">To draw a filled antialiased shape, first use the antialiased (aa*) version</span>
<span class="s0">of the function, and then use the filled (filled_*) version.</span>
<span class="s0">For example:</span>

<span class="s0">::</span>

   <span class="s0">col = (255, 0, 0)</span>
   <span class="s0">surf.fill((255, 255, 255))</span>
   <span class="s0">pygame.gfxdraw.aacircle(surf, x, y, 30, col)</span>
   <span class="s0">pygame.gfxdraw.filled_circle(surf, x, y, 30, col)</span>


<span class="s0">.. note::</span>
   <span class="s0">For threading, each of the functions releases the GIL during the C part of</span>
   <span class="s0">the call.</span>

<span class="s0">.. note::</span>
   <span class="s0">See the :mod:`pygame.draw` module for alternative draw methods.</span>
   <span class="s0">The ``pygame.gfxdraw`` module differs from the :mod:`pygame.draw` module in</span>
   <span class="s0">the API it uses and the different draw functions available.</span>
   <span class="s0">``pygame.gfxdraw`` wraps the primitives from the library called SDL_gfx,</span>
   <span class="s0">rather than using modified versions.</span>

<span class="s0">.. versionadded:: 1.9.0</span>

   <span class="s0">:meth:`box`.</span>

   <span class="s0">:param Surface surface: surface to draw on</span>

<span class="s0">.. function:: arc</span>

   <span class="s0">| :sl:`draw an arc`</span>
   <span class="s0">| :sg:`arc(surface, x, y, r, start_angle, stop_angle, color) -&gt; None`</span>

   <span class="s0">Draws an arc on the given surface. For an arc with its endpoints connected</span>
   <span class="s0">to its center use :meth:`pie`.</span>

   <span class="s0">The two angle arguments are given in degrees and indicate the start and stop</span>
   <span class="s0">positions of the arc. The arc is drawn in a clockwise direction from the</span>
   <span class="s0">``start_angle`` to the ``stop_angle``. If ``start_angle == stop_angle``,</span>
   <span class="s0">nothing will be drawn</span>

   <span class="s0">:param Surface surface: surface to draw on</span>
   <span class="s0">:param int x: x coordinate of the center of the arc</span>
   <span class="s0">:param int y: y coordinate of the center of the arc</span>
   <span class="s0">:param int r: radius of the arc</span>
   <span class="s0">:param int start_angle: start angle in degrees</span>
   <span class="s0">:param int stop_angle: stop angle in degrees</span>
   <span class="s0">:param color: color to draw with, the alpha value is optional if using a</span>
      <span class="s0">tuple ``(RGB[A])``</span>
   <span class="s0">:type color: Color or tuple(int, int, int, [int])</span>

   <span class="s0">:returns: ``None``</span>
   <span class="s0">:rtype: NoneType</span>

   <span class="s0">.. note::</span>
      <span class="s0">This function uses *degrees* while the :func:`pygame.draw.arc` function</span>
      <span class="s0">uses *radians*.</span>

   <span class="s0">.. ## pygame.gfxdraw.arc ##</span>

<span class="s0">.. function:: pie</span>

   <span class="s0">| :sl:`draw a pie`</span>
   <span class="s0">| :sg:`pie(surface, x, y, r, start_angle, stop_angle, color) -&gt; None`</span>

   <span class="s0">Draws an unfilled pie on the given surface. A pie is an :meth:`arc` with its</span>
   <span class="s0">endpoints connected to its center.</span>

   <span class="s0">The two angle arguments are given in degrees and indicate the start and stop</span>

   <span class="s0">a straight line will be drawn from the center position at the given angle,</span>
   <span class="s0">to a length of the radius.</span>

   <span class="s0">:param Surface surface: surface to draw on</span>
   <span class="s0">:param int x: x coordinate of the center of the pie</span>
   <span class="s0">:param int y: y coordinate of the center of the pie</span>
   <span class="s0">:param int r: radius of the pie</span>
   <span class="s0">:param int start_angle: start angle in degrees</span>
   <span class="s0">:param int stop_angle: stop angle in degrees</span>
   <span class="s0">:param color: color to draw with, the alpha value is optional if using a</span>
      <span class="s0">tuple ``(RGB[A])``</span>
   <span class="s0">:type color: Color or tuple(int, int, int, [int])</span>

   <span class="s0">:returns: ``None``</span>
   <span class="s0">:rtype: NoneType</span>

   <span class="s0">.. ## pygame.gfxdraw.pie ##</span>

   <span class="s0">| :sl:`draw a polygon`</span>
   <span class="s0">| :sg:`polygon(surface, points, color) -&gt; None`</span>

   <span class="s0">The adjacent coordinates in the ``points`` argument, as well as the first</span>

      <span class="s0">will be truncated)</span>
   <span class="s0">:type points: tuple(coordinate) or list(coordinate)</span>
   <span class="s0">:param color: color to draw with, the alpha value is optional if using a</span>
      <span class="s0">tuple ``(RGB[A])``</span>
   <span class="s0">:type color: Color or tuple(int, int, int, [int])</span>

   <span class="s0">:returns: ``None``</span>
   <span class="s0">:rtype: NoneType</span>

   <span class="s0">:raises ValueError: if ``len(points) &lt; 3`` (must have at least 3 points)</span>

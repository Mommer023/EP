<span class="s0">:mod:`pygame.key`</span>
<span class="s0">=================</span>

<span class="s0">.. module:: pygame.key</span>
   <span class="s0">:synopsis: pygame module to work with the keyboard</span>

<span class="s0">| :sl:`pygame module to work with the keyboard`</span>

<span class="s0">This module contains functions for dealing with the keyboard.</span>

<span class="s0">The :mod:`pygame.event` queue gets ``pygame.KEYDOWN`` and ``pygame.KEYUP``</span>
<span class="s0">events when the keyboard buttons are pressed and released. Both events have</span>
<span class="s0">``key`` and ``mod`` attributes.</span>

   <span class="s0">* ``key``: an :ref:`integer ID &lt;key-constants-label&gt;` representing every key</span>
     <span class="s0">on the keyboard</span>
   <span class="s0">* ``mod``: a bitmask of all the :ref:`modifier keys &lt;key-modifiers-label&gt;`</span>
     <span class="s0">that were in a pressed state when the event occurred</span>

<span class="s0">The ``pygame.KEYDOWN`` event has the additional attributes ``unicode`` and</span>
<span class="s0">``scancode``.</span>

   <span class="s0">* ``unicode``: a single character string that is the fully translated</span>
     <span class="s0">character entered, this takes into account the shift and composition keys</span>
   <span class="s0">* ``scancode``: the platform-specific key code, which could be different from</span>
     <span class="s0">keyboard to keyboard, but is useful for key selection of weird keys like</span>
     <span class="s0">the multimedia keys</span>

<span class="s0">.. versionadded:: 2.0.0</span>
    <span class="s0">The ``pygame.TEXTINPUT`` event is preferred to the ``unicode`` attribute</span>
    <span class="s0">of ``pygame.KEYDOWN``. The attribute ``text`` contains the input.</span>

<span class="s0">.. _key-modifiers-label:</span>

<span class="s0">The keyboard also has a list of modifier states (from :mod:`pygame.locals`) that</span>
<span class="s0">can be assembled by bitwise-ORing them together.</span>

<span class="s0">::</span>

      <span class="s0">pygame</span>
      <span class="s0">Constant      Description</span>
      <span class="s0">-------------------------</span>
      <span class="s0">KMOD_NONE     no modifier keys pressed</span>
      <span class="s0">KMOD_LSHIFT   left shift</span>
      <span class="s0">KMOD_RSHIFT   right shift</span>
      <span class="s0">KMOD_SHIFT    left shift or right shift or both</span>
      <span class="s0">KMOD_LCTRL    left control</span>
      <span class="s0">KMOD_RCTRL    right control</span>
      <span class="s0">KMOD_CTRL     left control or right control or both</span>
      <span class="s0">KMOD_LALT     left alt</span>
      <span class="s0">KMOD_RALT     right alt</span>
      <span class="s0">KMOD_ALT      left alt or right alt or both</span>
      <span class="s0">KMOD_LMETA    left meta</span>
      <span class="s0">KMOD_RMETA    right meta</span>
      <span class="s0">KMOD_META     left meta or right meta or both</span>
      <span class="s0">KMOD_CAPS     caps lock</span>
      <span class="s0">KMOD_NUM      num lock</span>
      <span class="s0">KMOD_MODE     AltGr</span>


<span class="s0">The modifier information is contained in the ``mod`` attribute of the</span>
<span class="s0">``pygame.KEYDOWN`` and ``pygame.KEYUP`` events. The ``mod`` attribute is a</span>
<span class="s0">bitmask of all the modifier keys that were in a pressed state when the event</span>
<span class="s0">occurred. The modifier information can be decoded using a bitwise AND (except</span>
<span class="s0">for ``KMOD_NONE``, which should be compared using equals ``==``). For example:</span>

<span class="s0">::</span>

    <span class="s0">for event in pygame.event.get():</span>
        <span class="s0">if event.type == pygame.KEYDOWN or event.type == pygame.KEYUP:</span>
            <span class="s0">if event.mod == pygame.KMOD_NONE:</span>
                <span class="s0">print('No modifier keys were in a pressed state when this '</span>
                      <span class="s0">'event occurred.')</span>
            <span class="s0">else:</span>
                <span class="s0">if event.mod &amp; pygame.KMOD_LSHIFT:</span>
                    <span class="s0">print('Left shift was in a pressed state when this event '</span>
                          <span class="s0">'occurred.')</span>
                <span class="s0">if event.mod &amp; pygame.KMOD_RSHIFT:</span>
                    <span class="s0">print('Right shift was in a pressed state when this event '</span>
                          <span class="s0">'occurred.')</span>
                <span class="s0">if event.mod &amp; pygame.KMOD_SHIFT:</span>
                    <span class="s0">print('Left shift or right shift or both were in a '</span>
                          <span class="s0">'pressed state when this event occurred.')</span>



<span class="s0">.. function:: get_focused</span>

   <span class="s0">| :sl:`true if the display is receiving keyboard input from the system`</span>
   <span class="s0">| :sg:`get_focused() -&gt; bool`</span>

   <span class="s0">Returns ``True`` when the display window has keyboard focus from the</span>
   <span class="s0">system. If the display needs to ensure it does not lose keyboard focus, it</span>
   <span class="s0">can use :func:`pygame.event.set_grab()` to grab all input.</span>

   <span class="s0">.. ## pygame.key.get_focused ##</span>

<span class="s0">.. function:: get_pressed</span>

   <span class="s0">| :sl:`get the state of all keyboard buttons`</span>
   <span class="s0">| :sg:`get_pressed() -&gt; bools`</span>

   <span class="s0">Returns a sequence of boolean values representing the state of every key on</span>
   <span class="s0">the keyboard. Use the key constant values to index the array. A ``True``</span>
   <span class="s0">value means that the button is pressed.</span>

   <span class="s0">.. note::</span>
      <span class="s0">Getting the list of pushed buttons with this function is not the proper</span>
      <span class="s0">way to handle text entry from the user. There is no way to know the order</span>
      <span class="s0">of keys pressed, and rapidly pushed keys can be completely unnoticed</span>
      <span class="s0">between two calls to ``pygame.key.get_pressed()``. There is also no way to</span>
      <span class="s0">translate these pushed keys into a fully translated character value. See</span>
      <span class="s0">the ``pygame.KEYDOWN`` events on the :mod:`pygame.event` queue for this</span>
      <span class="s0">functionality.</span>

   <span class="s0">.. ## pygame.key.get_pressed ##</span>

   <span class="s0">| :sl:`determine which modifier keys are being held`</span>

   <span class="s0">Returns a single integer representing a bitmask of all the modifier keys</span>

<span class="s0">.. function:: set_repeat</span>

   <span class="s0">| :sl:`control how held keys are repeated`</span>
   <span class="s0">| :sg:`set_repeat() -&gt; None`</span>
   <span class="s0">| :sg:`set_repeat(delay) -&gt; None`</span>
   <span class="s0">| :sg:`set_repeat(delay, interval) -&gt; None`</span>

   <span class="s0">When the keyboard repeat is enabled, keys that are held down will generate</span>
   <span class="s0">multiple ``pygame.KEYDOWN`` events. The ``delay`` parameter is the number of</span>
   <span class="s0">milliseconds before the first repeated ``pygame.KEYDOWN`` event will be sent.</span>
   <span class="s0">After that, another ``pygame.KEYDOWN`` event will be sent every ``interval``</span>
   <span class="s0">milliseconds. If a ``delay`` value is provided and an ``interval`` value is</span>
   <span class="s0">not provided or is 0, then the ``interval`` will be set to the same value as</span>
   <span class="s0">``delay``.</span>

   <span class="s0">To disable key repeat call this function with no arguments or with ``delay``</span>
   <span class="s0">set to 0.</span>

   <span class="s0">When pygame is initialized the key repeat is disabled.</span>

   <span class="s0">:raises ValueError: if ``delay`` or ``interval`` is &lt; 0</span>

   <span class="s0">.. versionchanged:: 2.0.0 A ``ValueError`` is now raised (instead of a</span>
      <span class="s0">``pygame.error``) if ``delay`` or ``interval`` is &lt; 0.</span>

   <span class="s0">.. ## pygame.key.set_repeat ##</span>

<span class="s0">.. function:: get_repeat</span>

   <span class="s0">| :sl:`see how held keys are repeated`</span>
   <span class="s0">| :sg:`get_repeat() -&gt; (delay, interval)`</span>

   <span class="s0">Get the ``delay`` and ``interval`` keyboard repeat values. Refer to</span>
   <span class="s0">:func:`pygame.key.set_repeat()` for a description of these values.</span>

   <span class="s0">.. versionadded:: 1.8</span>

   <span class="s0">.. ## pygame.key.get_repeat ##</span>

<span class="s0">.. function:: name</span>

   <span class="s0">| :sl:`get the name of a key identifier`</span>
   <span class="s0">| :sg:`name(key, use_compat=True) -&gt; str`</span>

   <span class="s0">Get the descriptive name of the button from a keyboard button id constant.</span>
   <span class="s0">Returns an empty string (``&quot;&quot;``) if the key is not found.</span>

   <span class="s0">If ``use_compat`` argument is ``True`` (which is the default), this function</span>
   <span class="s0">returns the legacy name of a key where applicable. The return value is</span>
   <span class="s0">expected to be the same across different pygame versions (provided the</span>
   <span class="s0">corresponding key constant exists and is unique). If the return value is</span>
   <span class="s0">passed to the ``key_code`` function, the original constant will be returned.</span>

   <span class="s0">**Experimental:** ``use_compat`` paramater still in development for testing and feedback. It may change.</span>
   <span class="s0">`Please leave use_compat feedback with authors &lt;https://github.com/pygame/pygame/pull/3312&gt;`_</span>

   <span class="s0">If this argument is ``False``, the returned name may be prettier to display</span>
   <span class="s0">and may cover a wider range of keys than with ``use_compat``, but there are</span>
   <span class="s0">no guarantees that this name will be the same across different pygame</span>
   <span class="s0">versions. If the name returned is passed to the ``key_code`` function, the</span>
   <span class="s0">original constant is returned back (this is an implementation detail which</span>
   <span class="s0">may change later, do not rely on this)</span>

   <span class="s0">.. versionchanged:: 2.1.3 Added ``use_compat`` argument and guaranteed API stability for it</span>

   <span class="s0">.. ## pygame.key.name ##</span>

<span class="s0">.. function:: key_code</span>

   <span class="s0">| :sl:`get the key identifier from a key name`</span>
   <span class="s0">| :sg:`key_code(name=string) -&gt; int`</span>

   <span class="s0">Get the key identifier code from the descriptive name of the key. This</span>
   <span class="s0">returns an integer matching one of the K_* keycodes. For example:</span>

   <span class="s0">::</span>

        <span class="s0">&gt;&gt;&gt; pygame.key.key_code(&quot;return&quot;) == pygame.K_RETURN</span>
        <span class="s0">True</span>
        <span class="s0">&gt;&gt;&gt; pygame.key.key_code(&quot;0&quot;) == pygame.K_0</span>
        <span class="s0">True</span>
        <span class="s0">&gt;&gt;&gt; pygame.key.key_code(&quot;space&quot;) == pygame.K_SPACE</span>
        <span class="s0">True</span>

   <span class="s0">:raises ValueError: if the key name is not known.</span>

   <span class="s0">.. versionadded:: 2.0.0</span>

   <span class="s0">.. ## pygame.key.key_code ##</span>

<span class="s0">.. function:: start_text_input</span>

   <span class="s0">| :sl:`start handling Unicode text input events`</span>
   <span class="s0">| :sg:`start_text_input() -&gt; None`</span>

   <span class="s0">Start receiving ``pygame.TEXTEDITING`` and ``pygame.TEXTINPUT``</span>
   <span class="s0">events. If applicable, show the on-screen keyboard or IME editor.</span>

   <span class="s0">For many languages, key presses will automatically generate a</span>
   <span class="s0">corresponding ``pygame.TEXTINPUT`` event. Special keys like</span>
   <span class="s0">escape or function keys, and certain key combinations will not</span>
   <span class="s0">generate ``pygame.TEXTINPUT`` events.</span>

   <span class="s0">In other languages, entering a single symbol may require multiple</span>
   <span class="s0">key presses, or a language-specific user interface. In this case,</span>
   <span class="s0">``pygame.TEXTINPUT`` events are preferable to ``pygame.KEYDOWN``</span>
   <span class="s0">events for text input.</span>

   <span class="s0">A ``pygame.TEXTEDITING`` event is received when an IME composition</span>
   <span class="s0">is started or changed. It contains the composition ``text``, ``length``,</span>
   <span class="s0">and editing ``start`` position within the composition (attributes</span>
   <span class="s0">``text``, ``length``, and ``start``, respectively).</span>
   <span class="s0">When the composition is committed (or non-IME input is received),</span>
   <span class="s0">a ``pygame.TEXTINPUT`` event is generated.</span>

   <span class="s0">Text input events handling is on by default.</span>

   <span class="s0">.. versionadded:: 2.0.0</span>

   <span class="s0">.. ## pygame.key.start_text_input ##</span>

<span class="s0">.. function:: stop_text_input</span>

   <span class="s0">| :sl:`stop handling Unicode text input events`</span>
   <span class="s0">| :sg:`stop_text_input() -&gt; None`</span>

   <span class="s0">Stop receiving ``pygame.TEXTEDITING`` and ``pygame.TEXTINPUT``</span>
   <span class="s0">events. If an on-screen keyboard or IME editor was shown with</span>
   <span class="s0">``pygame.key.start_text_input()``, hide it again.</span>

   <span class="s0">Text input events handling is on by default.</span>

   <span class="s0">To avoid triggering the IME editor or the on-screen keyboard</span>
   <span class="s0">when the user is holding down a key during gameplay, text input</span>

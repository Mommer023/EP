<span class="s0">.. module:: pygame._sdl2.controller</span>
   <span class="s0">:synopsis: pygame module to work with controllers</span>

<span class="s0">| :sl:`Pygame module to work with controllers.`</span>

<span class="s0">This module offers control over common controller types like the dualshock 4 or</span>
<span class="s0">the xbox 360 controllers: They have two analog sticks, two triggers, two shoulder buttons,</span>
<span class="s0">a dpad, 4 buttons on the side, 2 (or 3) buttons in the middle.</span>

<span class="s0">Pygame uses xbox controllers naming conventions (like a, b, x, y for buttons) but</span>
<span class="s0">they always refer to the same buttons. For example ``CONTROLLER_BUTTON_X`` is</span>
<span class="s0">always the leftmost button of the 4 buttons on the right.</span>

<span class="s0">Controllers can generate the following events::</span>

   <span class="s0">CONTROLLERAXISMOTION, CONTROLLERBUTTONDOWN, CONTROLLERBUTTONUP,</span>
   <span class="s0">CONTROLLERDEVICEREMAPPED, CONTROLLERDEVICEADDED, CONTROLLERDEVICEREMOVED</span>

<span class="s0">Additionally if pygame is built with SDL 2.0.14 or higher the following events can also be generated</span>
<span class="s0">(to get the version of sdl pygame is built with use :meth:`pygame.version.SDL`)::</span>

   <span class="s0">CONTROLLERTOUCHPADDOWN, CONTROLLERTOUCHPADMOTION, CONTROLLERTOUCHPADUP</span>

<span class="s0">These events can be enabled/disabled by :meth:`pygame._sdl2.controller.set_eventstate`</span>
<span class="s0">Note that controllers can generate joystick events as well. This function only toggles</span>
<span class="s0">events related to controllers.</span>

<span class="s0">.. note::</span>
   <span class="s0">See the :mod:`pygame.joystick` for a more versatile but more advanced api.</span>

<span class="s0">.. versionadded:: 2 This module requires SDL2.</span>

      <span class="s0">.. ## Contorller.get_mapping ##</span>

   <span class="s0">.. method:: set_mapping</span>

      <span class="s0">| :sl:`Assign a mapping to the controller`</span>
      <span class="s0">| :sg:`set_mapping(mapping) -&gt; int`</span>

      <span class="s0">Rebind buttons, axes, triggers and dpads. The mapping should be a </span>
      <span class="s0">dict containing all buttons, hats and axes. The easiest way to get this</span>
      <span class="s0">is to use the dict returned by :meth:`Controller.get_mapping`. To edit</span>
      <span class="s0">this mapping assign a value to the original button. The value of the</span>
      <span class="s0">dictionary must be a button, hat or axis represented in the following way:</span>

      <span class="s0">* For a button use: bX where X is the index of the button.</span>
      <span class="s0">* For a hat use: hX.Y where X is the index and the Y is the direction (up: 1, right: 2, down: 3, left: 4).</span>
      <span class="s0">* For an axis use: aX where x is the index of the axis.</span>

      <span class="s0">An example of mapping::</span>

         <span class="s0">mapping = controller.get_mapping() # Get current mapping</span>
         <span class="s0">mapping[&quot;a&quot;] = &quot;b3&quot; # Remap button a to y</span>
         <span class="s0">mapping[&quot;y&quot;] = &quot;b0&quot; # Remap button y to a</span>
         <span class="s0">controller.set_mapping(mapping) # Set the mapping</span>


      <span class="s0">The function will return 1 if a new mapping is added or 0 if an existing one is updated.</span>

      <span class="s0">.. versionchanged:: 2.0.2: Renamed from ``add_mapping`` to ``set_mapping``</span>
      <span class="s0">.. versionchanged:: 2.0.2: Argument type changed from ``str`` to ``dict``</span>

      <span class="s0">.. ## Contorller.set_mapping ##</span>

   <span class="s0">.. method:: rumble</span>

      <span class="s0">| :sl:`Start a rumbling effect`</span>
      <span class="s0">| :sg:`rumble(low_frequency, high_frequency, duration) -&gt; bool`</span>

      <span class="s0">Start a rumble effect on the controller, with the specified strength ranging</span>
      <span class="s0">from 0 to 1. Duration is length of the effect, in ms. Setting the duration</span>
      <span class="s0">to 0 will play the effect until another one overwrites it or</span>
      <span class="s0">:meth:`Controller.stop_rumble` is called. If an effect is already</span>
      <span class="s0">playing, then it will be overwritten.</span>

      <span class="s0">Returns True if the rumble was played successfully or False if the</span>
      <span class="s0">controller does not support it or :meth:`pygame.version.SDL` is below 2.0.9.</span>

      <span class="s0">.. versionadded:: 2.0.2</span>

      <span class="s0">.. ## Contorller.rumble ##</span>

   <span class="s0">.. method:: stop_rumble</span>

      <span class="s0">| :sl:`Stop any rumble effect playing`</span>
      <span class="s0">| :sg:`stop_rumble() -&gt; None`</span>

      <span class="s0">Stops any rumble effect playing on the controller. See</span>
      <span class="s0">:meth:`Controller.rumble` for more information.</span>

      <span class="s0">.. versionadded:: 2.0.2</span>

      <span class="s0">.. ## Contorller.stop_rumble ##</span>

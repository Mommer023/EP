<span class="s0">:mod:`pygame.sprite`</span>
<span class="s0">====================</span>

<span class="s0">.. module:: pygame.sprite</span>
   <span class="s0">:synopsis: pygame module with basic game object classes</span>

<span class="s0">| :sl:`pygame module with basic game object classes`</span>

<span class="s0">This module contains several simple classes to be used within games. There is</span>
<span class="s0">the main Sprite class and several Group classes that contain Sprites. The use</span>
<span class="s0">of these classes is entirely optional when using pygame. The classes are fairly</span>
<span class="s0">lightweight and only provide a starting place for the code that is common to</span>
<span class="s0">most games.</span>

<span class="s0">The Sprite class is intended to be used as a base class for the different types</span>
<span class="s0">of objects in the game. There is also a base Group class that simply stores</span>
<span class="s0">sprites. A game could create new types of Group classes that operate on</span>
<span class="s0">specially customized Sprite instances they contain.</span>

<span class="s0">The basic Sprite class can draw the Sprites it contains to a Surface. The</span>
<span class="s0">``Group.draw()`` method requires that each Sprite have a ``Surface.image``</span>
<span class="s0">attribute and a ``Surface.rect``. The ``Group.clear()`` method requires these</span>
<span class="s0">same attributes, and can be used to erase all the Sprites with background.</span>
<span class="s0">There are also more advanced Groups: ``pygame.sprite.RenderUpdates()`` and</span>
<span class="s0">``pygame.sprite.OrderedUpdates()``.</span>

<span class="s0">Lastly, this module contains several collision functions. These help find</span>
<span class="s0">sprites inside multiple groups that have intersecting bounding rectangles. To</span>
<span class="s0">find the collisions, the Sprites are required to have a ``Surface.rect``</span>
<span class="s0">attribute assigned.</span>

<span class="s0">The groups are designed for high efficiency in removing and adding Sprites to</span>
<span class="s0">them. They also allow cheap testing to see if a Sprite already exists in a</span>
<span class="s0">Group. A given Sprite can exist in any number of groups. A game could use some</span>
<span class="s0">groups to control object rendering, and a completely separate set of groups to</span>
<span class="s0">control interaction or player movement. Instead of adding type attributes or</span>
<span class="s0">bools to a derived Sprite class, consider keeping the Sprites inside organized</span>
<span class="s0">Groups. This will allow for easier lookup later in the game.</span>

<span class="s0">Sprites and Groups manage their relationships with the ``add()`` and</span>
<span class="s0">``remove()`` methods. These methods can accept a single or multiple targets for</span>
<span class="s0">membership. The default initializers for these classes also takes a single or</span>
<span class="s0">list of targets for initial membership. It is safe to repeatedly add and remove</span>
<span class="s0">the same Sprite from a Group.</span>

<span class="s0">While it is possible to design sprite and group classes that don't derive from</span>
<span class="s0">the Sprite and AbstractGroup classes below, it is strongly recommended that you</span>
<span class="s0">extend those when you add a Sprite or Group class.</span>

<span class="s0">Sprites are not thread safe. So lock them yourself if using threads.</span>

<span class="s0">.. class:: Sprite</span>

   <span class="s0">| :sl:`Simple base class for visible game objects.`</span>
   <span class="s0">| :sg:`Sprite(*groups) -&gt; Sprite`</span>

   <span class="s0">The base class for visible game objects. Derived classes will want to</span>
   <span class="s0">override the ``Sprite.update()`` and assign a ``Sprite.image`` and</span>
   <span class="s0">``Sprite.rect`` attributes. The initializer can accept any number of Group</span>
   <span class="s0">instances to be added to.</span>

   <span class="s0">When subclassing the Sprite, be sure to call the base initializer before</span>
   <span class="s0">adding the Sprite to Groups. For example:</span>

   <span class="s0">.. code-block:: python</span>
   
       <span class="s0">class Block(pygame.sprite.Sprite):</span>
            
           <span class="s0"># Constructor. Pass in the color of the block, </span>
           <span class="s0"># and its x and y position</span>
           <span class="s0">def __init__(self, color, width, height):</span>
              <span class="s0"># Call the parent class (Sprite) constructor</span>
              <span class="s0">pygame.sprite.Sprite.__init__(self) </span>
        
              <span class="s0"># Create an image of the block, and fill it with a color.</span>
              <span class="s0"># This could also be an image loaded from the disk.</span>
              <span class="s0">self.image = pygame.Surface([width, height])</span>
              <span class="s0">self.image.fill(color)</span>
        
              <span class="s0"># Fetch the rectangle object that has the dimensions of the image</span>
              <span class="s0"># Update the position of this object by setting the values of rect.x and rect.y</span>
              <span class="s0">self.rect = self.image.get_rect()   </span>
      
   <span class="s0">.. method:: update</span>

      <span class="s0">| :sl:`method to control sprite behavior`</span>
      <span class="s0">| :sg:`update(*args, **kwargs) -&gt; None`</span>

      <span class="s0">The default implementation of this method does nothing; it's just a</span>
      <span class="s0">convenient &quot;hook&quot; that you can override. This method is called by</span>
      <span class="s0">``Group.update()`` with whatever arguments you give it.</span>

      <span class="s0">There is no need to use this method if not using the convenience method</span>
      <span class="s0">by the same name in the Group class.</span>

      <span class="s0">.. ## Sprite.remove ##</span>

   <span class="s0">.. method:: kill</span>

      <span class="s0">| :sl:`remove the Sprite from all Groups`</span>
      <span class="s0">| :sg:`kill() -&gt; None`</span>

      <span class="s0">The Sprite is removed from all the Groups that contain it. This won't</span>
      <span class="s0">change anything about the state of the Sprite. It is possible to continue</span>
      <span class="s0">to use the Sprite after this method has been called, including adding it</span>

<span class="s0">.. class:: DirtySprite</span>

   <span class="s0">| :sl:`A subclass of Sprite with more attributes and features.`</span>
   <span class="s0">| :sg:`DirtySprite(*groups) -&gt; DirtySprite`</span>

   <span class="s0">Extra DirtySprite attributes with their default values:</span>

   <span class="s0">dirty = 1</span>

   <span class="s0">::</span>

       <span class="s0">if set to 1, it is repainted and then set to 0 again</span>
       <span class="s0">if set to 2 then it is always dirty ( repainted each frame,</span>
       <span class="s0">flag is not reset)</span>
       <span class="s0">0 means that it is not dirty and therefore not repainted again</span>

   <span class="s0">blendmode = 0</span>

   <span class="s0">::</span>

       <span class="s0">its the special_flags argument of blit, blendmodes</span>

   <span class="s0">source_rect = None</span>

   <span class="s0">::</span>

       <span class="s0">source rect to use, remember that it is relative to</span>
       <span class="s0">topleft (0,0) of self.image</span>

   <span class="s0">visible = 1</span>

   <span class="s0">::</span>

       <span class="s0">normally 1, if set to 0 it will not be repainted</span>
       <span class="s0">(you must set it dirty too to be erased from screen)</span>

   <span class="s0">layer = 0</span>

   <span class="s0">::</span>

       <span class="s0">(READONLY value, it is read when adding it to the</span>
       <span class="s0">LayeredDirty, for details see doc of LayeredDirty)</span>

<span class="s0">.. class:: Group</span>

   <span class="s0">| :sl:`A container class to hold and manage multiple Sprite objects.`</span>
   <span class="s0">| :sg:`Group(*sprites) -&gt; Group`</span>

   <span class="s0">A simple container for Sprite objects. This class can be inherited to create</span>
   <span class="s0">containers with more specific behaviors. The constructor takes any number of</span>
   <span class="s0">Sprite arguments to add to the Group. The group supports the following</span>
   <span class="s0">standard Python operations:</span>

   <span class="s0">::</span>

       <span class="s0">in      test if a Sprite is contained</span>
       <span class="s0">len     the number of Sprites contained</span>
       <span class="s0">bool    test if any Sprites are contained</span>
       <span class="s0">iter    iterate through all the Sprites</span>

   <span class="s0">The Sprites in the Group are ordered only on python 3.6 and higher.</span>
   <span class="s0">Below python 3.6 drawing and iterating over the Sprites is in no particular order.</span>

   <span class="s0">.. method:: sprites</span>

      <span class="s0">| :sl:`list of the Sprites this Group contains`</span>
      <span class="s0">| :sg:`sprites() -&gt; sprite_list`</span>

      <span class="s0">Return a list of all the Sprites this group contains. You can also get an</span>
      <span class="s0">iterator from the group, but you cannot iterate over a Group while</span>
      <span class="s0">modifying it.</span>

      <span class="s0">.. ## Group.sprites ##</span>

   <span class="s0">.. method:: copy</span>

      <span class="s0">| :sl:`duplicate the Group`</span>
      <span class="s0">| :sg:`copy() -&gt; Group`</span>

      <span class="s0">Creates a new Group with all the same Sprites as the original. If you</span>
      <span class="s0">have subclassed Group, the new object will have the same (sub-)class as</span>
      <span class="s0">the original. This only works if the derived class's constructor takes</span>
      <span class="s0">the same arguments as the Group class's.</span>

      <span class="s0">.. ## Group.copy ##</span>

   <span class="s0">.. method:: add</span>

      <span class="s0">| :sl:`add Sprites to this Group`</span>
      <span class="s0">| :sg:`add(*sprites) -&gt; None`</span>

      <span class="s0">Add any number of Sprites to this Group. This will only add Sprites that</span>
      <span class="s0">are not already members of the Group.</span>

      <span class="s0">Each sprite argument can also be a iterator containing Sprites.</span>

      <span class="s0">.. ## Group.add ##</span>

   <span class="s0">.. method:: remove</span>

      <span class="s0">| :sl:`remove Sprites from the Group`</span>
      <span class="s0">| :sg:`remove(*sprites) -&gt; None`</span>

      <span class="s0">Remove any number of Sprites from the Group. This will only remove</span>
      <span class="s0">Sprites that are already members of the Group.</span>

      <span class="s0">Each sprite argument can also be a iterator containing Sprites.</span>

      <span class="s0">.. ## Group.remove ##</span>

   <span class="s0">.. method:: has</span>

      <span class="s0">| :sl:`test if a Group contains Sprites`</span>
      <span class="s0">| :sg:`has(*sprites) -&gt; bool`</span>

      <span class="s0">Return True if the Group contains all of the given sprites. This is</span>
      <span class="s0">similar to using the &quot;in&quot; operator on the Group (&quot;if sprite in group:</span>
      <span class="s0">...&quot;), which tests if a single Sprite belongs to a Group.</span>

      <span class="s0">Each sprite argument can also be a iterator containing Sprites.</span>

      <span class="s0">.. ## Group.has ##</span>

   <span class="s0">.. method:: update</span>

      <span class="s0">| :sl:`call the update method on contained Sprites`</span>
      <span class="s0">| :sg:`update(*args, **kwargs) -&gt; None`</span>

      <span class="s0">Calls the ``update()`` method on all Sprites in the Group. The base</span>
      <span class="s0">Sprite class has an update method that takes any number of arguments and</span>
      <span class="s0">does nothing. The arguments passed to ``Group.update()`` will be passed</span>
      <span class="s0">to each Sprite.</span>

      <span class="s0">There is no way to get the return value from the ``Sprite.update()``</span>
      <span class="s0">methods.</span>

      <span class="s0">.. ## Group.update ##</span>

   <span class="s0">.. method:: draw</span>

      <span class="s0">| :sl:`blit the Sprite images`</span>
      <span class="s0">| :sg:`draw(Surface) -&gt; List[Rect]`</span>

      <span class="s0">Draws the contained Sprites to the Surface argument. This uses the</span>
      <span class="s0">``Sprite.image`` attribute for the source surface, and ``Sprite.rect``</span>
      <span class="s0">for the position.</span>

      <span class="s0">The Group does not keep sprites in any order, so the draw order is</span>
      <span class="s0">arbitrary.</span>

      <span class="s0">.. ## Group.draw ##</span>

   <span class="s0">.. method:: clear</span>

      <span class="s0">| :sl:`draw a background over the Sprites`</span>
      <span class="s0">| :sg:`clear(Surface_dest, background) -&gt; None`</span>

      <span class="s0">Erases the Sprites used in the last ``Group.draw()`` call. The</span>
      <span class="s0">destination Surface is cleared by filling the drawn Sprite positions with</span>
      <span class="s0">the background.</span>

      <span class="s0">The background is usually a Surface image the same dimensions as the</span>
      <span class="s0">destination Surface. However, it can also be a callback function that</span>
      <span class="s0">takes two arguments; the destination Surface and an area to clear. The</span>
      <span class="s0">background callback function will be called several times each clear.</span>

      <span class="s0">Here is an example callback that will clear the Sprites with solid red:</span>

      <span class="s0">::</span>

          <span class="s0">def clear_callback(surf, rect):</span>
              <span class="s0">color = 255, 0, 0</span>
              <span class="s0">surf.fill(color, rect)</span>

      <span class="s0">.. ## Group.clear ##</span>

<span class="s0">.. class:: RenderUpdates</span>

   <span class="s0">| :sl:`Group sub-class that tracks dirty updates.`</span>
   <span class="s0">| :sg:`RenderUpdates(*sprites) -&gt; RenderUpdates`</span>

   <span class="s0">This class is derived from ``pygame.sprite.Group()``. It has an extended</span>
   <span class="s0">``draw()`` method that tracks the changed areas of the screen.</span>

   <span class="s0">.. method:: draw</span>

      <span class="s0">| :sl:`blit the Sprite images and track changed areas`</span>
      <span class="s0">| :sg:`draw(surface) -&gt; Rect_list`</span>

      <span class="s0">Draws all the Sprites to the surface, the same as ``Group.draw()``. This</span>
      <span class="s0">method also returns a list of Rectangular areas on the screen that have</span>
      <span class="s0">been changed. The returned changes include areas of the screen that have</span>
      <span class="s0">been affected by previous ``Group.clear()`` calls.</span>

      <span class="s0">The returned Rect list should be passed to ``pygame.display.update()``.</span>
      <span class="s0">This will help performance on software driven display modes. This type of</span>
      <span class="s0">updating is usually only helpful on destinations with non-animating</span>
      <span class="s0">backgrounds.</span>

<span class="s0">.. function:: OrderedUpdates</span>

   <span class="s0">| :sl:`RenderUpdates sub-class that draws Sprites in order of addition.`</span>
   <span class="s0">| :sg:`OrderedUpdates(*sprites) -&gt; OrderedUpdates`</span>

   <span class="s0">This class derives from ``pygame.sprite.RenderUpdates()``. It maintains the</span>

<span class="s0">.. class:: LayeredUpdates</span>

   <span class="s0">| :sl:`LayeredUpdates is a sprite group that handles layers and draws like OrderedUpdates.`</span>
   <span class="s0">| :sg:`LayeredUpdates(*sprites, **kwargs) -&gt; LayeredUpdates`</span>

   <span class="s0">This group is fully compatible with :class:`pygame.sprite.Sprite`.</span>

   <span class="s0">You can set the default layer through kwargs using 'default_layer' and an</span>
   <span class="s0">integer for the layer. The default layer is 0.</span>

   <span class="s0">If the sprite you add has an attribute _layer then that layer will be used.</span>
   <span class="s0">If the \**kwarg contains 'layer' then the sprites passed will be added to</span>
   <span class="s0">that layer (overriding the ``sprite.layer`` attribute). If neither sprite</span>
   <span class="s0">has attribute layer nor \**kwarg then the default layer is used to add the</span>
   <span class="s0">sprites.</span>

   <span class="s0">.. versionadded:: 1.8</span>

   <span class="s0">.. method:: add</span>

      <span class="s0">| :sl:`add a sprite or sequence of sprites to a group`</span>
      <span class="s0">| :sg:`add(*sprites, **kwargs) -&gt; None`</span>

      <span class="s0">If the ``sprite(s)`` have an attribute layer then that is used for the</span>
      <span class="s0">layer. If \**kwargs contains 'layer' then the ``sprite(s)`` will be added</span>
      <span class="s0">to that argument (overriding the sprite layer attribute). If neither is</span>
      <span class="s0">passed then the ``sprite(s)`` will be added to the default layer.</span>

<span class="s0">.. function:: spritecollide</span>

   <span class="s0">| :sl:`Find sprites in a group that intersect another sprite.`</span>
   <span class="s0">| :sg:`spritecollide(sprite, group, dokill, collided = None) -&gt; Sprite_list`</span>

   <span class="s0">Return a list containing all Sprites in a Group that intersect with another</span>
   <span class="s0">Sprite. Intersection is determined by comparing the ``Sprite.rect``</span>
   <span class="s0">attribute of each Sprite.</span>

   <span class="s0">The dokill argument is a bool. If set to True, all Sprites that collide will</span>
   <span class="s0">be removed from the Group.</span>

   <span class="s0">The collided argument is a callback function used to calculate if two</span>
   <span class="s0">sprites are colliding. it should take two sprites as values, and return a</span>
   <span class="s0">bool value indicating if they are colliding. If collided is not passed, all</span>
   <span class="s0">sprites must have a &quot;rect&quot; value, which is a rectangle of the sprite area,</span>
   <span class="s0">which will be used to calculate the collision.</span>

   <span class="s0">collided callables:</span>

   <span class="s0">::</span>

       <span class="s0">collide_rect, collide_rect_ratio, collide_circle,</span>
       <span class="s0">collide_circle_ratio, collide_mask</span>

   <span class="s0">Example:</span>
   
   <span class="s0">.. code-block:: python</span>

    <span class="s0"># See if the Sprite block has collided with anything in the Group block_list</span>
    <span class="s0"># The True flag will remove the sprite in block_list</span>
    <span class="s0">blocks_hit_list = pygame.sprite.spritecollide(player, block_list, True)  </span>
     
    <span class="s0"># Check the list of colliding sprites, and add one to the score for each one</span>
    <span class="s0">for block in blocks_hit_list:</span>
        <span class="s0">score +=1</span>

<span class="s0">.. function:: collide_circle</span>

   <span class="s0">| :sl:`Collision detection between two sprites, using circles.`</span>
   <span class="s0">| :sg:`collide_circle(left, right) -&gt; bool`</span>

   <span class="s0">Tests for collision between two sprites, by testing to see if two circles</span>

   <span class="s0">| :sl:`Collision detection between two sprites, using circles scaled to a ratio.`</span>
   <span class="s0">| :sg:`collide_circle_ratio(ratio) -&gt; collided_callable`</span>

   <span class="s0">A callable class that checks for collisions between two sprites, using a</span>
   <span class="s0">scaled version of the sprites radius.</span>

   <span class="s0">Is created with a floating point ratio, the instance is then intended to be</span>

<span class="s0">.. function:: collide_mask</span>

   <span class="s0">| :sl:`Collision detection between two sprites, using masks.`</span>
   <span class="s0">| :sg:`collide_mask(sprite1, sprite2) -&gt; (int, int)`</span>
   <span class="s0">| :sg:`collide_mask(sprite1, sprite2) -&gt; None`</span>

   <span class="s0">Tests for collision between two sprites, by testing if their bitmasks</span>
   <span class="s0">overlap (uses :func:`pygame.mask.Mask.overlap`). If the sprites have a</span>
   <span class="s0">``mask`` attribute, it is used as the mask, otherwise a mask is created from</span>
   <span class="s0">the sprite's ``image`` (uses :func:`pygame.mask.from_surface`). Sprites must</span>
   <span class="s0">have a ``rect`` attribute; the ``mask`` attribute is optional.</span>

   <span class="s0">The first point of collision between the masks is returned. The collision</span>
   <span class="s0">point is offset from ``sprite1``'s mask's topleft corner (which is always</span>
   <span class="s0">(0, 0)). The collision point is a position within the mask and is not</span>
   <span class="s0">related to the actual screen position of ``sprite1``.</span>

   <span class="s0">This function is intended to be passed as a ``collided`` callback function</span>
   <span class="s0">to the group collide functions (see :meth:`spritecollide`,</span>
   <span class="s0">:meth:`groupcollide`, :meth:`spritecollideany`).</span>

   <span class="s0">.. note::</span>
      <span class="s0">To increase performance, create and set a ``mask`` attribute for all</span>
      <span class="s0">sprites that will use this function to check for collisions. Otherwise,</span>
      <span class="s0">each time this function is called it will create new masks.</span>

   <span class="s0">.. note::</span>
      <span class="s0">A new mask needs to be recreated each time a sprite's image is changed</span>
      <span class="s0">(e.g. if a new image is used or the existing image is rotated).</span>

   <span class="s0">::</span>

      <span class="s0"># Example of mask creation for a sprite.</span>
      <span class="s0">sprite.mask = pygame.mask.from_surface(sprite.image)</span>

   <span class="s0">:returns: first point of collision between the masks or ``None`` if no</span>
      <span class="s0">collision</span>
   <span class="s0">:rtype: tuple(int, int) or NoneType</span>

   <span class="s0">.. versionadded:: 1.8.0</span>

   <span class="s0">.. ## pygame.sprite.collide_mask ##</span>

<span class="s0">.. function:: groupcollide</span>

   <span class="s0">| :sl:`Find all sprites that collide between two groups.`</span>
   <span class="s0">| :sg:`groupcollide(group1, group2, dokill1, dokill2, collided = None) -&gt; Sprite_dict`</span>

   <span class="s0">This will find collisions between all the Sprites in two groups.</span>

   <span class="s0">The collided argument is a callback function used to calculate if two sprites are</span>

   <span class="s0">sprites must have a &quot;rect&quot; value, which is a rectangle of the sprite area,</span>
   <span class="s0">which will be used to calculate the collision.</span>
